# Business Research Skill

Scripted workflow for turning a raw idea into a validated, cited, visually-rich HTML report served on the public web.

## Skill Contract

**Inputs:**
- A business/product/startup idea or question.
- Optional constraints from the user: geography, target customer, funding stage, timeline, budget.

**Outputs:**
- `/var/www/html/researches/<slug>-<ulid>/index.html` — self-contained HTML
- Public URL: `http://168.110.213.104/researches/<slug>-<ulid>/`
- Tailscale URL: `http://100.124.60.57/researches/<slug>-<ulid>/`

**Behavior:**
- Always clarify scope before burning tool calls.
- Always cite sources with name + date + a direct quote or number.
- Never fabricate currency figures. If you must estimate, mark it `[ESTIMATED]` and show assumptions.
- Produce a ULID for every artifact. Use `ulid` CLI (installed on this host) or generate via Python `ulid-py`.

---

## ULID Generation

```bash
ulid
# or: python3 -c "import ulid; print(ulid.new().str)"
```

## Slug Generation

Derive from the user's idea description:
- Lowercase
- ASCII only, replace non-alphanumeric with `-`, collapse repeats
- Max 60 chars
- Remove leading/trailing `-`

Example: `ai-coffee-subscription-ramadan-bundle` → `ai-coffee-subscription-ramadan-bundle`

## Output Directory

```
/var/www/html/researches/<slug>-<ulid>/
├── index.html          # self-contained, no external CDN deps
└── README.md           # one-liner summary for future reference (optional)
```

nginx routing (already configured):
```
location /researches/ {
    alias /var/www/html/researches/;
    try_files $uri $uri/ /researches/$uri/index.html;
    autoindex on;
}
```

If `/researches/` nginx location is missing, stop and ask to create it — do not publish without a serving path.

## HTML Report Schema

Report name: `<slug>-<ulid>& #8212; Business Research`

Sections (in order):
1. **Executive Summary** (150 words max, single paragraph)
2. **Idea Definition** (the user's exact ask, paraphrased)
3. **Market Landscape**
   - TAM / SAM / SOM with sources
   - Existing vs new market classification + rationale
   - Growth rate / CAGR if available
4. **SWOT Analysis** (2×2 grid or 4-column table)
5. **Competitor Analysis**
   - Direct competitors (2–5)
   - Indirect / substitute competitors (2–3)
   - For each: name, offering, pricing ceiling, differentiability, defensibility
6. **Unit Economics Sketch**
   - Cost structure (fixed + variable per unit)
   - Revenue model (tier, transaction, subscription, hybrid)
   - Break-even volume rough estimate
7. **Expansion Playbook**
   - Pillar 1: Penetration
   - Pillar 2: Product stretch
   - Pillar 3: New market segment
   - Pillar 4: M&A or partnerships (if Stage C/D fundable)
8. **Regulatory & Technology Constraints** (relevant to the sector)
9. **Sources** (numbered, with name + date + direct quote or granularity)
10. **Confidence & Open Questions** (overall confidence %, top 3 remaining unknowns)

## HTML Style Requirements

- Self-contained (no CDN, no external CSS, no external images needed)
- Inline all CSS in `<style>`
- Color palette: navy `#1a2942`, teal `#0d9488`, amber `#d97706`, white/cream
- Typography: system fonts stack (Inter, -apple-system, Segoe UI, Helvetica, Arial)
- Layout: max-width 1120px, centered
- Print-friendly with `@media print`
- Sections separated by a teal rule with spacing
- SWOT as a 2×2 grid with distinct background colors per cell
- Data tables with zebra striping and right-aligned numbers
- **Make it look like a boutique strategy memo, not a Bootstrap tutorial**
- Embed a small "Generated by Hermes Researcher" footer with timestamp

## Workflow (Follow Exactly)

### Step 0: Clarify (MANDATORY — do not skip)

Before any tool calls, ask the user scoping questions until you have >= 5 of these fields:

| Field           | Why it matters                       |
|-----------------|--------------------------------------|
| Geography       | Market data is country-specific      |
| Target customer | Narrows competitor set               |
| Unit of value   | SaaS seat vs one-off vs subscription |
| Stage           | Idea vs MVP vs revenue               |
| Timeline        | Determines evidence recency priority |
| Differentiation | Prevents generic competitor search   |

**You must ask at least 2 open-ended questions.**
If the user is short: "I don't know, just research generally" → proceed with global/default assumptions and flag them in the report.

### Step 1: Reconnaissance

Use `web` tools (search + extraction) to build a source list.

Focus first on:
- Second-degree startup databases: Crunchbase, PitchBook previews, AngelList, Tracxn free tier
- Market reports: Gartner (free summaries), Statista (free previews), IBISWorld abstracts, World Bank, IMF, OEC
- Regulatory: official gov sites, EU/OECD/WHO guidelines
- Competitor websites + recent press

Log each source:
```
SOURCE:<name>|<date>|<url>|<1-line finding>
```

Stop at 25 sources or 3 zero-yield consecutive searches — whichever comes first.

### Step 2: Synthesize

For each section of the HTML Schema:
- State the finding in ≤ 3 sentences
- Attach source number from log

Keep track of:
- Sourced numbers → directly quoted
- Range estimates → [ESTIMATED] shown
- Pure analyst opinion → flagged [OPINION] with source

### Step 3: Risk Checks

Before writing HTML, ask yourself:
- Did I fabricate any number? If yes, either find real data or mark [ESTIMATED] with assumptions.
- Is the SWOT internally consistent? (e.g., "market growing" should show up in Opportunities, not Threat)
- Are competitors spelled correctly and linked to their official site when feasible?
- Is the break-even logic arithmetically consistent?

### Step 4: HTML Generation

Use `execute_code` with Python to:

1. Generate a ULID: `import ulid; print(ulid.new().str)`
2. Create the directory: `/var/www/html/researches/<slug>-<ulid>/`
3. Write `index.html` with all sections inlined

Write clean, readable Python — no minification. The HTML will be visible to the user so make it maintainable.

### Step 5: Nginx Publishing

By default `/var/www/html/researches/` is already covered by the existing nginx config block. Verify by:
```bash
curl -s http://127.0.0.1/researches/<slug>-<ulid>/ | head -3
```

If you get a 404, stop and ask to add a location block rather than publishing blind.

### Step 6: Handoff

Return `public URL` and `tailscale URL`. Include a 4-bullet TL;DR summary of the research so the user has the gist without opening the link.

## Error Handling

| Failure | Response |
|---------|----------|
| ULID generator missing | `pip install ulid-py` in a venv or fall back to a timestamped token format: `yyyymmdd-hhmmss-<random4>` |
| No nginx coverage | Stop, surface the gap, do not write HTML to an unserved directory |
| Conflicting market data | Show both sources, date-stamp both, let user decide which to trust |
| Zero reliable sources for a section | Write the section with "No reliable source found; recommend primary research." |
| User asks for live numbers while you are still writing | Pause, present what you have, allow user to pick a direction |
